================================================================================
PROJECT CONVENTIONS
================================================================================

Package Naming:
  - Pattern: odogwudozilla.year<YYYY>.day<D>
  - Each puzzle gets its own package
  - Main class naming: <PuzzleTitle>AOC<YYYY>Day<D>.java

Class Structure:
  - Each solution is a standalone Java class with main() method
  - Includes JavaDoc with puzzle description and URL
  - Reads input from resource files (never hardcoded)
  - Solves both Part 1 and Part 2 (if applicable)

Resource Files:
  - Puzzle descriptions stored in day<D>_puzzle_description.txt
  - Puzzle input data stored in day<D>_puzzle_data.txt
  - Kept separate to allow easy reference and modification

Configuration:
  - aoc_challenge_config.json    - Defines available years/days
  - solutions_database.json      - Tracks solved puzzles and results

================================================================================
KEY FILES & THEIR PURPOSES
================================================================================

Main.java
  Purpose: Application entry point
  Role: Orchestrates the overall application flow

PuzzleRandomizer.java
  Purpose: Intelligent puzzle selection utility
  Role: Randomly selects unsolved puzzles from available set
  Features:
    - Reads aoc_challenge_config.json for available puzzles
    - Reads solutions_database.json for completed puzzles
    - Excludes future-dated puzzles
    - Returns random unsolved puzzle in format: YYYY,D

aoc_challenge_config.json
  Purpose: Central configuration for puzzle availability
  Contents:
    - Available years and date ranges
    - Total days per year
    - Date when puzzles became available

solutions_database.json
  Purpose: Persistent storage of puzzle solutions
  Contents:
    - Puzzle metadata (title, link, year, day)
    - Solution results (Part 1 and Part 2 answers)
    - Organized by year then day

README.md files
  Purpose: Documentation at year and project levels
  Locations:
    - Root: src/main/java/odogwudozilla/README.md (project overview)
    - Year: src/main/java/odogwudozilla/year<YYYY>/README.md (year details)

================================================================================
WORKFLOW OVERVIEW
================================================================================

1. Random Puzzle Selection
   - Run: ./gradlew randomPuzzle
   - Output: YYYY,D format
   - Creates new year/day package if needed

2. Setup Phase
   - Create package: odogwudozilla.year<YYYY>.day<D>
   - Create resources: <YYYY>/day<D>/
   - Create empty files: day<D>_puzzle_description.txt
                        day<D>_puzzle_data.txt

3. Data Collection
   - Paste puzzle description to day<D>_puzzle_description.txt
   - Paste puzzle input to day<D>_puzzle_data.txt

4. Solution Development
   - Create class: <PuzzleName>AOC<YYYY>Day<D>.java
   - Implement Part 1 solution
   - Implement Part 2 solution (if applicable)
   - Include JavaDoc and comments

5. Database & Documentation Update
   - Update solutions_database.json with results
   - Update/create year README.md
   - Update main README.md if new year
   - Update project_structure.txt if new year added or structure changed

6. Commit
   - Stage: git add -A
   - Commit: git commit -m "Add AOC <Year> Day <Day> solution: <Title> - <approach>"
